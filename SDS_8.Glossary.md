# 8. Glossary

| 이름 | 설명 |
|------|------|
| React | 본 프로젝트의 클라이언트를 개발하기 위해 사용되는 JavaScript 라이브러리. 사용자 인터페이스를 구축하기 위해 컴포넌트 기반의 방식을 사용한다. |
| Spring Boot | 본 프로젝트의 서버를 개발하기 위해 사용되는 Java 프레임워크. API 서버 구축, 데이터베이스 연동, 비지니스 로직 처리 등을 담당한다. |
| API | 클라이언트와 서버 간의 데이터 송수신을 위한 표준화된 통신 규약. RESTful API 방식으로 구현되며 HTTP 메서드를 사용한다. |
| Entity | 데이터베이스의 테이블과 일대일로 매핑되는 Java 객체. 실제 데이터를 저장하는 구조를 의미한다. |
| Logging | 시스템에서 발생하는 모든 이벤트 오류, 중요한 작업 등을 기록하는 것. |
| Redirection | 사용자에게서 요청이 들어오면 다른 페이지나 URL로 자동으로 이동시키는 것. |
| Database | 사용자 정보, 스터디룸 정보, 매칭 기록 등 시스템의 모든 데이터를 저장하고 관리하는 시스템. 본 프로젝트에서는 관계형 데이터베이스 MySQL을 사용한다. |
| URL | 웹상의 자원(페이지, API, 이미지 등)의 주소. 본 프로젝트의 친구 초대 기능에서는 특정 스터디룸이나 사용자를 초대하기 위한 고유한 URL이 생성된다. |
| Toggle | 어떤 상태를 ON/OFF로 전환하는 기능. 본 프로젝트의 음성/화상 연결, 알람 설정 등 특정 기능을 켜고 끄는 데 사용된다. |
| DTO | 패키지 간 데이터를 전달하기 위한 객체. 필요한 데이터만 담아서 전송해 데이터 보안과 효율성을 제공한다. |
| DTD | XML 문서의 구조와 규칙을 정의하는 파일. 본 프로젝트에서는 주로 XML 기반 설정 파일을 검증할 때 사용할 수 있다. |
| Interface | Java에서 클래스가 구현해야 할 메서드들을 정의한다. Repository, Service 등이 Interface로 정의되어 구현 클래스에서 구현된다. |
| Repository | 데이터베이스에 접근하여 생성, 수정, 삭제, 조회하는 패키지. Service 패키지에서 필요한 데이터 접근 요청을 받아 처리한다. |
| JWT | 사용자 인증을 위한 토큰 기반 인증 방식. 사용자가 로그인 성공 시 서버에서 JWT를 생성하여 클라이언트에게 전달하고, 이후 API 요청 시 이 토큰을 사용하여 사용자를 인증한다. |
| Controller | 클라이언트의 HTTP 요청에 해당하는 Service로 전달하고, Service의 반환값을 JSON 형태로 클라이언트에게 응답하는 패키지. |
| HttpOnly Cookie | JavaScript 코드에서 접근할 수 없도록 보호되는 쿠키. |
| Swagger | RESTful API를 정의, 설명, 생성, 시각화, 사용하기 위한 표준. |
